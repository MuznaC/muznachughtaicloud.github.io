Linux Software Management System

In order to update the Linux machine using the package manager and install the AWS Command Line Interface (AWS CLI) we first need to use SSH to connect to an Amazon Linux EC2 instance

Firstly, we download the pem file, then we open a terminal window, and change directory cd to the directory where the pem file was downloaded. to access the file we will run "cd ~/Downloads" command.

To change the permissions on the key to be read-only, we run the command: "chmod 400 project.pem"

Next, we return to the EC2 Console and select Instances. Check the box next to the instance you want to connect to and in the Description tab copy the IPv4 Public IP value and type "yes" after using this command: "ssh -i labsuser.pem ec2-user@10.0.10.163

Secondly, we update our Linux machine. To query repositories for available updates, enter "sudo yum -y check-update" and to apply security-related updates, enter "sudo yum update --security". To update packages, we enter "sudo yum -y upgrade".

To view the install of httpd and view the history of updates, enter "sudo yum install httpd -y"

Then we downgrade a package that has been updated through the yum package manager by doing the following steps:

  • Using the yum history to list what has been installed and updated
  • Rolling back to the most recent updates in the history list

To view the history of updates, enter "sudo yum history list" . In the output, under the ID column, we make a note of the number for EC2 ... to use in the history info task.

To view the most recent set of updates, enter sudo yum history info 1

Lastly, we install the AWS CLI on Red Hat Linux. To verify that Python is installed, we enter "python3 --version"

To see if the pip package manager is already installed. we enter this command:"pip3 --version"

In order to install the AWS CLI, download the installation file using the curl command.

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"

The above command write the downloaded file to the current directory with the local name awscliv2.zip.

Then we unzip the installer: "unzip awscliv2.zip"

Run the install program: "sudo ./aws/install"

To verify that the AWS CLI is now working, enter the following command and press Enter. aws help

At the : prompt, enter q to exit. And we make copy of aws_access_key_id and aws_secret_access_key

Lastly, we go back to terminal and run the command "aws configure"

At the prompts, enter the following information:

  • For the AWS Access Key ID, leave blank and press Enter.
  • For the AWS Secret Access Key, leave blank and press Enter.
  • For the Default region name, enter us-west-2 and press Enter.
  • For the Default output format, enter json and press Enter.
  • For the Default output format, enter json and press Enter.
  • After the information is entered, the appropriate credential files are created automatically.
  • To open the credential file, enter the command sudo nano ~/.aws/credentials

Press ctrl + O to save and Press ctrl + X to exit the file

Write the following command with instance id:

"aws ec2 describe-instance-attribute --instance-id i-1234567890abcdefg --attribute instanceType"
